# Pages Management

The Pages module allows users to create and manage website pages without writing code.

Pages are built using a **component-based system**, where sections (components) are added from the Component Panel.

---

## 📄 Pages List Overview

All created pages are displayed in a table view inside the CMS.

Each page includes:

- Page Name  
- Permalink  
- Status (Draft / Publish)  
- Actions (Edit / Delete)  

### 🖼️ Pages Table View

> List of all created pages with management actions.

![Pages List](/images/pages-list.png)

---

## ➕ Creating a New Page

To create a new page:

1. Click **Add Page**
2. Enter page details
3. Configure layout and SEO settings
4. Save or publish the page

### 🖼️ Add Page Form

> Page creation interface inside CMS.

![Add Page](/images/add-page.png)

---

## Adding UI Blocks (Components)

After creating a page, users can build the page content using **UI Blocks**.

UI Blocks are reusable components created by developers and available inside the CMS.

When editing a page, users will see the **Add UI Block** option.

From here they can:

- Open the UI Block / Component Panel  
- Select available components (Hero, About, Services, Gallery, etc.)  
- Add components to the page  
- Arrange the order of components  
- Configure content for each component  

👉 Each UI Block represents a section of the website page.

![Add UI Block](/images/add-ui-block.png)

## 🔗 Permalinks

Each page uses a permalink to define its URL.

Example:

```
about-us
contact
services
```

⚠️ Rules:
- Must be unique  
- No leading slash (/)  

---

## 🧭 Breadcrumb Settings

Breadcrumbs help users navigate pages easily.

Options include:

- Enable / Disable breadcrumb  
- Custom breadcrumb text  
- Background image upload  

### 🖼️ Breadcrumb Settings

![Breadcrumb Settings](/images/breadcrumb.png)

---

## 🎨 Header & Footer Settings

Each page can override layout settings:

- Header: Fixed / Sticky  
- Footer: Show / Hide  

These settings are applied per page.

---

## 🚀 Publish Settings

Pages support three actions:

- Save (Draft mode)  
- Save & Exit  
- Preview (frontend preview only)  

### 🖼️ Publish Options

![Publish Options](/images/publish-options.png)

---

## 📌 Page Status

- Draft → Not visible on website  
- Published → Visible on website  

---

## 🔍 SEO Settings

SEO options allow full search engine optimization.

Fields include:

- SEO Title (max 60 characters)  
- SEO Description (max 160 characters)  
- SEO Image upload  
- Index / No Index option  

---

## 🧠 Indexing Behavior

- Index → Page is visible on search engines  
- No Index → Page is hidden from search engines  

### 🖼️ SEO Settings Panel

![SEO Settings](/images/seo-settings.png)

---

## 🚀 Summary

The Pages system gives users full control over creating SEO-optimized and dynamic website pages.

Pages are built using **reusable components (sections)** from the Component Panel, allowing flexible page design without code.
